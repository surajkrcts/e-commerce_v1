E-Commerce Project — Documentation

Minimal Spring Boot backend: REST APIs for products, carts, orders and users.

Overview Prerequisites Setup Configuration Architecture Implementation Testing Deployment User Docs

Purpose & Goals

A compact Spring Boot service for learning and small deployments. Exposes REST endpoints and demonstrates layered design, error handling, and unit tests.

Scope

  • REST APIs for catalog, cart, orders, and user management.

Prerequisites

  • Java 21
  • Maven 3.6+
  • Git
  • IDE such as IntelliJ IDEA 2025.1.3
  • Database : MySQL 8.0.4+

Setup

Clone the repository and build with Maven, then run the application.

git clone https://github.com/surajkrcts/e-commerce_v1
cd e-commerce_v1
mvn clean install
mvn spring-boot:run

Configuration

Edit application properties for database and server settings.

src/main/resources/application.properties
  • spring.datasource.url ={database URL}
  • spring.datasource.username ={database Username}
  • spring.datasource.password ={database Password}
  • server.port ={server port}
  • Use profiles and environment variables for secrets

Architecture & Design

Layered architecture: Controller → Service → Repository. Uses a relational database and DTOs for API responses.

  • Base API path: /api/v1/
  • Typical endpoints: products, orders, users
  • Standard HTTP status codes and consistent JSON error responses

Implementation Details

  • Package layout: controller, service, repository, model (entities & DTOs)
  • Naming: PascalCase for classes, camelCase for methods, dto (requests and responses), exceptions end with Exception
  • Dependencies: Spring Boot starters, JPA, JUnit + Mockito, Lombok, Swagger

Testing

Unit tests with Mockito and jUnit

Deployment & Operations

Build and run the jar for production. CI should run tests and produce artifacts.

mvn clean package
java -jar target/*.jar --spring.profiles.active=prod

User Documentation

Quick start: ensure DB config, run the app, then call APIs at http://localhost:{server.port}/api/v1/

Troubleshooting

  • Startup failures: check DB connection and logs
  • 404 on endpoint: verify controller paths and base path
  • Failing tests: run tests locally and inspect stack traces
  • Cors failure: make sure @CrossOrigin is added on controller class
